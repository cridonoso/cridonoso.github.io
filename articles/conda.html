Instalacion Miniconda + VirtualEnv

En este tutorial explicare como instalar Anaconda (o su version liviana miniconda) en Linux y Windows. Ademas configuraremos un ambiente virtual para trabajar de manera aislada, lo cual nos permitira:

  • Evitar romper el kernel de nuestro computador (o sistema operativo) al instalar una libreria
  • Configurar multiples espacios de trabajo para nuestros proyectos. De esta forma, podemos tener ambientes con distintas versiones y dependencias dependiendo de los requerimientos del proyecto.
  • Exportar nuestro ambiente virtual e instalarlo en otra maquina que tenga anaconda

Instalacion

En esta seccion explicare como instalar Miniconda en Windows y Linux. Asi que puedes saltarte la seccion que no coincida con tu sistema operativo.
Por cierto, debes asegurarte de no tener instalado anaconda en tu computador. Para saber si ya tienes anaconda instalado abre una terminal y coloca conda --version. En windows puedes ver si tienes instalado Anaconda Prompt

Podemos verificar que anaconda quedo correctamente instalado abriendo Anaconda Powershell Prompt desde el menu de inicio y escribiendo: conda --version. Utilizare Powershell porque dispone de los mismos comandos bash de Linux.

Linux

En linux debemos descargar el instalador desde este link. El archivo tiene extension .sh lo cual quiere decir que debemos ejecutarlo por bash. Para instalar debemos abrir una terminal y colocar: 
sudo bash Miniconda3-latest-Linux-x86_64.sh
Y apretamos enter hasta que te aparezca el directorio donde quieres colocar conda. Personalmente me gusta utilizar la ruta /opt/miniconda3. Si no cambias la ruta quedara almacenado en el /home.

Una vez hecho eso nos va a preguntar si quieres agregar conda al .init de la terminal. Yo siempre coloco que No.

En la ultima parte debemos configurar el PATH para acceder a conda en nuestra terminal. Para esto vamos abrir el archivo oculto .bashrc con privilegios de superusario 
sudo nano ~/.bashrc
Y al final del archivo agregamos la siguiente linea: 
export PATH=/opt/miniconda3/bin:$PATH
donde /opt/miniconda3 es el directorio que elejiste para guardar miniconda.

Listo! Ahora cerramos nuestra terminal, abrimos una nueva y colocamos conda --version para ver que esta todo en orden.


Creando un Ambiente Virtual

Por defecto conda dispone del ambiente (base) el cual trae Python y algunos paquetes muy basicos.

Sin embargo nosotros queremos crear nuestro propio entorno. Para ello abrimos Anaconda Powershell Prompt desde el inicio, o bien una nueva terminal en Linux.

Para crear un nuevo entorno: 
conda create -n python pandas numpy
En la sentencia anterior estamos utilizando la funcion create de conda para crear un nuevo ambiente virtual llamado MiPrimerVE. Notese que -n es el flag que permite ponerle un nombre a nuestro entorno (el string que venga despues del flag sera considerado el nombre).
Los demas string son librerias que podemos pasar como argumento para que sean instaladas durante el proceso de creacion.

Automaticamente nos preguntara si queremos instalar todas las dependencias necesarias. Aqui ponemos y

Una vez terminado el proceso, conda nos dice el comando necesario para entrar a nuestro entorno. 

conda activate python
Si no te funciona este comando, prueba con: 

source activate python
Para salir puedes utilizar conda deactivate o bien source deactivate

Deberias ver un (python) con el nombre del entorno.

Instalando una libreria

Para instalar una libreria, primero debemos entrar a nuestro entorno: 
conda activate python
Luego utilizamos: 
conda install LIBRERIA
Por ejemplo si queremos instalar Jupyter Notebook puedes escribir: 
conda install notebook
Nos preguntara por las dependencias a instalar

damos y y listo.

En el caso particular de jupyter notebook debemos, desde nuestra terminal o prompt, colocar: 
jupyter notebook
Automaticamente se abrira nuestro navegador con Jupyter notebook en el directorio donde hayamos ejecutado el comando.
