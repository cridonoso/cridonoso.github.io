Github Basics

You have probably arrived here because of me saying something like:

Hey, look at my blog, I posted my first article about Git and Github, very funny, very useful.
If you are reading this just for fun, please give me your feedback. I will really appreciate your comments to improve this blog. If you are here for need, I will convince you that Git and GitHub are extremely necessary when working in groups or alone. But first of all:

What is Git and Github

Git is a Distributed version control software. Distributed because each programmer within the project has its own version of the code running locally. They work to contribute the same master code by patching different project versions. Git provides us tools for tracking projects if something gets out of control, which is very useful for large projects. Personally, I am always changing my code, implementing very cool features but with incredible destructive powers.

On the other hand, Github is where we store the project for free (unless you need privacy, private ownership is paid). For free and private repositories, check out this alternative. In my view, Github is like hacking social media. People create profiles and share their codes with the community. Many projects are uploaded to Github; Just take an in-depth look at your favorite framework's source code!

Using Git + Github, you can
  1. Clone your repository wherever you want
  2. Manage project versions
  3. Go to any point in the timeline of your project
  4. Filter changes on the master code by creating pull requests
  5. Share your code/project with other people socializing knowledge
I hope I have convinced you. Remember, every new tool has its learning curve. Once we get used to it, everything becomes easier. Let's start!
